<What>

The Anthropic backend uses the native Messages API (`/messages` endpoint) with a full translation layer between OpenAI Chat Completions format and Anthropic's wire format. It supports prompt caching, extended thinking (extended reasoning), tool use, and vision.

<How>

### Configuration

```yaml
# ~/.genesis/config.yaml
model: claude-sonnet-4-20250514
backend: anthropic
```

API key is read from `ANTHROPIC_API_KEY`. The default base URL is `https://api.anthropic.com/v1`. Requests go to the `/messages` endpoint.

### Request headers

The client automatically sets:
- `x-api-key: {ANTHROPIC_API_KEY}`
- `anthropic-version: 2023-06-01`
- `anthropic-beta: token-efficient-tools-2025-02-19,fine-grained-tool-streaming-2025-05-14`

### Format translation

The translation layer in `anthropic_types.rs` converts the OpenAI `ChatCompletionRequest` to an `AnthropicRequest`:

- The `system` message is extracted from the messages array and placed in the top-level `system` field
- User/assistant text content is mapped to Anthropic content blocks
- `tool_calls` (OpenAI format) â†’ `tool_use` content blocks
- Tool results â†’ `tool_result` content blocks
- Response: `AnthropicResponse` â†’ `ChatCompletionResponse`

### Prompt caching

For the `anthropic` backend (and `openrouter`), `inject_cache_control` automatically adds `cache_control: {"type": "ephemeral"}` to:

1. All system messages â€” converting them from a string to a content-block array with a `cache_control` marker
2. The last tool in the tools list â€” placing the cache marker at the tool level

This enables Anthropic's prompt caching feature, which can significantly reduce cost and latency for repeated conversations with the same system prompt and tool list.

### Extended thinking

When a `ThinkingConfig` is present in the request, the translation layer adds an `AnthropicThinking` block to the request:

```rust
AnthropicThinking {
    thinking_type: "enabled",
    budget_tokens: config.budget_tokens,
}
```

Thinking content blocks returned by the model are mapped back to the response and stored in `provider_metadata` on the message.

### Streaming

Anthropic's SSE event types differ from OpenAI's. The streaming path translates Anthropic events to `ChatCompletionChunk` objects:
- `content_block_delta` â†’ text delta or tool argument delta
- `message_delta` â†’ finish reason and usage
- `message_stop` â†’ stream end

</How>

<Why>

**Native API over OpenAI-compatible proxy.** Using Anthropic's native `/messages` endpoint rather than a compatibility layer gives access to features like extended thinking, fine-grained tool streaming, and token-efficient tools that may not be exposed through the OpenAI-compatible path.

**Automatic `cache_control` injection.** Injecting cache markers transparently means callers do not need to know about prompt caching â€” it is applied to every Anthropic and OpenRouter request automatically, reducing costs for users with large system prompts and many tools without any configuration.

**Beta header inclusion.** The `anthropic-beta` header enables beta features (token-efficient-tools, fine-grained-tool-streaming) that are harmless when not applicable and reduce costs/latency when they are. Including them globally avoids per-feature opt-ins.

</Why>
