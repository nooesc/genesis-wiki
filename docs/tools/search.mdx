<What>

## What Does search_files Do?

`search_files` searches file contents for a regex pattern, returning each matching line with its filename and line number. It uses ripgrep (`rg`) when available for fast, parallel searching, and falls back to `grep -rn` on systems where ripgrep is not installed. The `search_engine` metadata field reports which backend was used.

Optional filters let you narrow results by file type (e.g., `"rust"`, `"python"`), glob pattern (e.g., `"*.toml"`), and surrounding context lines. Results are capped at 100 matches and truncated at 64 KiB.

</What>

<How>

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `pattern` | `string` | Yes | -- | Regex pattern to search for |
| `path` | `string` | No | `.` | Directory to search within |
| `case_insensitive` | `string` | No | `"false"` | Set `"true"` for case-insensitive matching |
| `file_type` | `string` | No | -- | ripgrep file type filter, e.g. `"rust"`, `"py"` (rg only) |
| `glob` | `string` | No | -- | Glob pattern to filter files, e.g. `"*.toml"` (rg only) |
| `context_lines` | `string` | No | `"0"` | Number of surrounding context lines to include (rg only) |

## Example

```json
{
  "name": "search_files",
  "arguments": {
    "pattern": "fn run\\(",
    "path": "/home/user/my-project",
    "file_type": "rust",
    "context_lines": "2"
  }
}
```

**Response:**
```
src/main.rs:12:fn run(args: &Args) -> Result<()> {
src/lib.rs:45:fn run(config: Config) -> anyhow::Result<()> {
```

</How>

<Why>

## ripgrep vs grep Fallback

ripgrep availability is checked once at startup using `which rg` and cached via `OnceLock`. When rg is present, the tool passes `--line-number --no-heading --color never --max-count 100` for clean, parseable output. The `file_type` and `glob` parameters map directly to rg's `--type` and `--glob` flags, which are not available in the grep fallback. `context_lines` maps to rg's `-C` flag.

The grep fallback uses `-rn --include=* -m 100` and ignores the rg-specific parameters. The `search_engine` metadata field (`"rg"` or `"grep"`) lets callers know which backend ran.

</Why>
