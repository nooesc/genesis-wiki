<What>

## What is `genesis context`?

The `context` command manages the `.genesis/context.md` file in your current project directory. This file is automatically injected into Eve's system prompt whenever you run `genesis chat` or `genesis run` from within the project, giving her project-specific context without you having to explain it every session.

A context file typically contains information like: the purpose of the project, the tech stack, coding conventions, important file locations, known constraints, and any instructions specific to how you want Eve to behave in this codebase. It is the per-project equivalent of global memories.

</What>

<How>

## Subcommands

| Subcommand | Description |
|------------|-------------|
| `show` | Show the current project's context file |
| `init` | Create a `.genesis/context.md` template in the current directory |
| `edit` | Open the context file in `$EDITOR` |

## Examples

**Check if a context file exists and view it:**
```bash
genesis context show
```

**Create a starter template:**
```bash
genesis context init
```

**Open in your editor:**
```bash
genesis context edit
```

## Example Context File

```markdown
# Project: My Web App

## Stack
- Backend: Rust (Axum), PostgreSQL
- Frontend: React, TypeScript, Tailwind CSS
- Infrastructure: Docker, GitHub Actions

## Conventions
- All new code must have unit tests
- Use `snake_case` for Rust, `camelCase` for TypeScript
- Commits follow Conventional Commits format

## Key Locations
- API handlers: `src/handlers/`
- Database migrations: `migrations/`
- Frontend: `web/src/`

## Known Constraints
- The database schema is pinned to PostgreSQL 15
- Do not use `unwrap()` in production code paths
```

</How>

<Why>

## Context vs Global Memories

Global memories (managed by `genesis memory`) are facts about you — your preferences, recurring context, personal information. The project context file is facts about the current project — its architecture, conventions, and codebase-specific rules. The two are injected separately: memories go into a `## Memories` section of the system prompt, while the context file is included under a `## Project Context` section.

## Security Scanning

Context files loaded from `.genesis/context.md` pass through the context security scanner in `genesis-core` before injection. The scanner checks for prompt injection patterns — invisible unicode, instruction overrides, data exfiltration attempts, role manipulation. This matters because context files in shared repositories could be modified by other contributors, intentionally or unintentionally introducing content that tries to redirect Eve's behaviour. The scanner can be configured to warn or block depending on `runtime.context_security` setting.

## Automatic Detection

When you start `genesis chat` or `genesis run`, the `SystemPromptBuilder` walks up the directory tree from the current working directory looking for a `.genesis/context.md` file. The first one found is loaded. This means you can have different context files for different subdirectories of a monorepo, and the correct one will be used based on where you invoke Genesis.

</Why>
