<What>

## What is `genesis eval validate`?

The `validate` command checks every trajectory JSON file in a directory for structural integrity. It verifies that each file can be parsed as valid JSON, that it conforms to the expected trajectory schema (required fields present, types correct, steps well-formed), and that there are no orphaned or malformed step records. Invalid files are reported with a description of the problem. Use `--remove` to automatically delete files that fail validation, cleaning up a dataset before further processing.

</What>

<How>

## Arguments and Options

| Arg/Flag | Type | Required | Description |
|----------|------|----------|-------------|
| `<DIR>` | `string` | Yes | Directory containing trajectory JSON files |
| `--recursive` | `bool` | No | Recursively scan nested directories |
| `--remove` | `bool` | No | Delete invalid files |

## Examples

```bash
# Validate all trajectories and report problems
genesis eval validate ./trajectories

# Validate and delete invalid files
genesis eval validate ./trajectories --remove

# Recursively validate nested directories
genesis eval validate ./trajectories --recursive

# Validate before running the rest of a pipeline
genesis eval validate ./raw --remove && genesis eval filter ./raw --output ./clean --success-only
```

</How>

<Why>

## Implementation Notes

Validation catches two classes of problems: parse errors (the file is not valid JSON) and schema errors (the JSON is valid but missing required fields or has wrong types). Schema validation checks for the presence of `session_id`, `steps`, and `outcome` at minimum. Step records are checked for required `action_type` and `timestamp` fields. The `--remove` flag is intentionally separate from the default behavior so you must opt in to destructive cleanup. The `eval pipeline` command can run validation as its first step via `--validate`.

</Why>
