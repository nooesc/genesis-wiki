<What>

## What is `genesis eval stats`?

The `stats` command computes numerical dataset statistics across a directory of trajectory JSON files. It reports counts, totals, and distributions: total trajectories, success/failure breakdown, step count distribution (min, max, mean, median), token usage totals and averages, model distribution, and tool usage frequency. Unlike `summarize`, which focuses on replay quality, `stats` focuses on the quantitative shape of the dataset — useful for understanding coverage, balance, and size before training.

</What>

<How>

## Arguments and Options

| Arg/Flag | Type | Required | Description |
|----------|------|----------|-------------|
| `<DIR>` | `string` | Yes | Directory containing trajectory JSON files |
| `--recursive` | `bool` | No | Recursively scan nested directories for trajectory JSON files |
| `--model <MODEL>` | `string` | No | Only include trajectories for this model |
| `--tag <TAG>` | `string` | No | Only include trajectories tagged with this value |
| `--tool <TOOL>` | `string` | No | Only include trajectories that used this tool |
| `--failures-only` | `bool` | No | Only include trajectories whose outcome is failure |

## Examples

```bash
# Compute stats for all trajectories in a directory
genesis eval stats ./trajectories

# Stats for a specific model's trajectories only
genesis eval stats ./trajectories --model claude-sonnet-4-6

# Stats for trajectories tagged "file-ops"
genesis eval stats ./trajectories --tag file-ops
```

</How>

<Why>

## Implementation Notes

`stats` and `summarize` are complementary: `summarize` gives you a qualitative replay analysis (warnings, failures, tool patterns), while `stats` gives you quantitative dataset metrics (counts, distributions, totals). The same filter flags appear in both commands so you can compare filtered subsets consistently. Step count and token distributions are reported as min/max/mean/median to give a sense of dataset variance without printing every value.

</Why>
