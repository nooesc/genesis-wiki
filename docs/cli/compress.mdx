<What>

## What is `genesis compress`?

The `compress` command reduces the size of a trajectory JSON file by collapsing or summarizing its messages, then optionally converts it to a different format. Trajectories generated by `genesis batch` can be verbose — they include full tool call outputs, intermediate reasoning, and metadata. Compression makes them smaller and more focused for training.

Three compression levels are available (`light`, `medium`, `heavy`) and two output formats beyond the native JSON: `chatml` (for fine-tuning pipelines that expect ChatML JSONL) and `sharegpt` (for the ShareGPT training format). The `--training` flag activates the training compressor, which protects the first and last turns (which carry the most signal) and summarizes the middle.

</What>

<How>

## Options

| Flag | Description |
|------|-------------|
| `--input <FILE>` | Input trajectory JSON file (required) |
| `--output <FILE>` | Output file path; writes to stdout when omitted |
| `--level <LEVEL>` | Compression level: `light`, `medium`, or `heavy` |
| `--format <FORMAT>` | Output format: `json`, `sharegpt`, or `chatml` |
| `--training` | Use the training compressor (protects first/last turns, summarizes middle) |

## Examples

**Light compression to stdout:**
```bash
genesis compress --input trajectory.json --level light
```

**Heavy compression to a new file:**
```bash
genesis compress --input trajectory.json --output compressed.json --level heavy
```

**Training compressor with ChatML output:**
```bash
genesis compress --input trajectory.json --training --format chatml --output training.jsonl
```

**Pipe to another tool:**
```bash
genesis compress --input trajectory.json --level medium | jq '.messages | length'
```

</How>

<Why>

## Compression Strategy

Standard compression (`--level`) works by truncating or dropping intermediate tool call outputs and assistant reasoning that is not load-bearing for the final response. `light` drops only the most verbose tool outputs; `heavy` aggressively reduces intermediate turns.

The training compressor (`--training`) uses a different strategy: it preserves the first and last turns exactly (the opening context and the final answer are the most signal-dense), and replaces the middle turns with a summary. This is `TrajectoryCompressor` in `genesis-core`, designed for producing training data where the model should learn from the task setup and outcome rather than every intermediate tool call.

The `--format` flag handles the conversion after compression, so you can compress and convert in a single pass rather than piping `genesis compress` into `genesis eval convert`.

</Why>
