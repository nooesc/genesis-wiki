<ApiEndpoint method="GET" path="/subagents/{id}" auth={true}>

<What>

Fetch the execution record for a single subagent by its ID. Subagents are child agent executions spawned by the `spawn_subagent` tool when Eve delegates a task to a parallel or specialised agent instance.

</What>

<How>

## Response

```json
{
  "id": "sub-abc123",
  "parent_session_id": "api-1741234567890-1",
  "prompt": "Analyse the security vulnerabilities in the authentication module",
  "result": "Found 2 potential issues: missing rate limiting on /login and weak session token entropy.",
  "status": "completed",
  "created_at": "2026-03-15 10:32:00",
  "completed_at": "2026-03-15 10:33:45"
}
```

| Field | Description |
|-------|-------------|
| `id` | Unique subagent execution ID |
| `parent_session_id` | The session that spawned this subagent |
| `prompt` | The task delegated to the subagent |
| `result` | The subagent's output, or `null` if still running |
| `status` | `pending`, `running`, `completed`, or `failed` |
| `created_at` | When the subagent was spawned |
| `completed_at` | When execution finished, or `null` |

Returns `404` if no record exists for the given ID.

</How>

<Why>

Inspect a specific subagent's result when debugging a parent session's multi-agent workflow. The subagent ID is returned by the `spawn_subagent` tool call and also appears in `GET /sessions/{id}/subagents`.

</Why>

</ApiEndpoint>

<ApiEndpoint method="GET" path="/sessions/{id}/subagents" auth={true}>

<What>

List all subagents spawned during a session. Returns an empty array if the session ran no subagents.

</What>

<How>

## Response

```json
{
  "session_id": "api-1741234567890-1",
  "subagents": [
    {
      "id": "sub-abc123",
      "parent_session_id": "api-1741234567890-1",
      "prompt": "Analyse security vulnerabilities in the auth module",
      "result": "Found 2 potential issues...",
      "status": "completed",
      "created_at": "2026-03-15 10:32:00",
      "completed_at": "2026-03-15 10:33:45"
    },
    {
      "id": "sub-def456",
      "parent_session_id": "api-1741234567890-1",
      "prompt": "Generate unit tests for the authentication module",
      "result": null,
      "status": "running",
      "created_at": "2026-03-15 10:34:00",
      "completed_at": null
    }
  ],
  "count": 2
}
```

</How>

<Why>

When a session runs multi-agent workflows, the parent session's result may not make sense without understanding what tasks were delegated and what results they produced. This endpoint gives you the full picture of all parallel execution that happened within a session.

</Why>

</ApiEndpoint>
